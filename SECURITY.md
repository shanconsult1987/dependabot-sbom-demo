# Security Policy

## 📌 Supported Versions

We actively support the following versions of this project:

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | ✅ Supported       |
| < 1.0   | ❌ Not supported   |

---

## 🚨 Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly:

- 📧 Email: security@yourcompany.com
- 🔒 Do NOT open public issues for security vulnerabilities
- 📝 Include:
  - Description of the issue
  - Steps to reproduce
  - Impact assessment
  - Suggested fix (if available)

---

## ⏱️ Response SLA (Enterprise Standard)

| Severity   | Initial Response | Resolution Target |
|------------|----------------|------------------|
| Critical   | 24 hours       | 72 hours         |
| High       | 48 hours       | 5 days           |
| Medium     | 72 hours       | 10 days          |
| Low        | 5 days         | 30 days          |

---

## 🔐 Security Measures in This Repository

This project uses:

- ✅ Dependabot (dependency vulnerability management)
- ✅ CodeQL (static code analysis)
- ✅ SBOM (Software Bill of Materials)
- ✅ GitHub Secret Scanning

---

## 🔄 Vulnerability Management Process

1. Vulnerability reported
2. Triage by security team
3. Reproduction and validation
4. Fix development
5. Security patch release
6. Public disclosure (if applicable)

---

## 🛡️ Disclosure Policy

- We follow **responsible disclosure**
- Public disclosure only after fix is available
- Credit given to reporters (if desired)

---

## ⚖️ Compliance & Standards

This project aligns with:

- OWASP Top 10
- NIST Secure Software Development Framework (SSDF)
- ISO/IEC 27001 best practices

---

## 🚫 Out of Scope

The following are not considered security vulnerabilities:

- Denial of service via brute force
- Missing best practices without exploit
- Social engineering attacks

---

## 🙏 Acknowledgements

We appreciate responsible disclosure and contributions from the security community.
